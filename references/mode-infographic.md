# 信息大图模式（infographic）

高密度信息图生成。先采集数据，再构建知识框架，最后用 AI 图像生成输出 2K/4K 信息大图。

所有 prompt 模板统一存储在 `config/prompts.yaml` 的 `infographic` 节。

## 一、5 种风格速查

| # | 风格 ID | 名称 | 关键词 | 视觉特征 | 适用场景 |
|---|---------|------|--------|---------|---------|
| 1 | tech_blueprint | 坐标蓝图 | 蓝图/坐标/实验室/技术 | 坐标网格+工程标注+蓝色调 | 技术、架构、系统类 |
| 2 | retro_pop | 复古波普 | 波普/复古/70s | 粗线条+高饱和CMYK+半调圆点 | 流行文化、趋势、对比类 |
| 3 | scrapbook | 手帐拼贴 | 手帐/拼贴/证据板 | 软木板+图钉+纸胶带+撕纸 | 历史、故事、情感类 |
| 4 | clay_doodle | 陶土手绘 | 陶土/手绘/暖色 | 手绘不规则线条+赤陶暖色 | 人文、生活、成长类 |
| 5 | vector | 矢量插图 | 矢量/扁平/几何 | 扁平单线+几何简化+纯色块 | 概念、理论、抽象类 |

**默认风格**：用户未指定时，根据主题自动匹配（匹配不了则用 #1 坐标蓝图）。

## 二、数据采集与过滤

### 2.1 数据采集

使用 Web 搜索工具（优先 brave-search，降级 web-search-prime）采集主题相关数据：

- 搜索 3-5 组关键词，覆盖不同角度
- 每组取前 5 条结果
- 提取具体数据点、案例、对比数据、时间线节点

### 2.2 数据过滤（三原则）

| 过滤器 | 标准 | 示例 |
|--------|------|------|
| **实用** Practical | 可指导行动，有操作价值 | "每天写 500 字" > "写作很重要" |
| **稀缺** Rare | 非常识，读者不太可能知道 | "GPT-4 用了 1.8T 参数" > "AI 很强大" |
| **共鸣** Resonant | 有情感冲击或认知颠覆 | "95% 的新技能 6 个月后只剩 20%" > "要持续学习" |

### 2.3 模块化拆分

将过滤后的数据拆分为 3-7 个模块，每个模块 6-7 个子要点。

## 三、工作流

### Step 1：采集主题

- 从用户输入中提取主题关键词
- 如果用户直接给了数据/框架，跳过采集，直接进入 Step 4

### Step 2：Web 搜索采集数据

- 用 brave-search 或 web-search-prime 搜索主题
- 搜索 3-5 组不同关键词
- 收集具体数据点、案例、对比

### Step 3：过滤与模块化

- 按三原则（实用/稀缺/共鸣）过滤数据
- 拆分为 3-7 个模块，每模块 6-7 子要点
- 输出框架供用户确认

### Step 4：呈现框架，等待确认

```
信息大图框架：{主题}
模块数：X | 风格建议：{风格名}
---
模块 1：...
模块 2：...
---
确认生成？或需要调整？
```

### Step 5：用户确认 → 选风格 → 从 prompts.yaml 获取 prompt

- 用户确认框架后，选择风格（或自动匹配）
- 读取 `config/prompts.yaml` 的 `infographic` 节
- 将主题和模块要点融入对应风格的 prompt 模板，替换 `{TOPIC}`

### Step 6：调用 generate.py

```bash
# 快捷方式（--style 自动读取比例和模板）：
python scripts/generate.py \
  --config config/engine.json \
  --style retro_pop --prompt "[包含模块要点的主题描述]" \
  --out "output/主题/infographic.jpg" \
  --aspect-ratio "3:4"

# 手动方式：
python scripts/generate.py \
  --config config/engine.json \
  --prompt "[从 prompts.yaml 读取并替换 {TOPIC} 后的完整 prompt]" \
  --out "output/主题/infographic.jpg" \
  --aspect-ratio "3:4"
```

### Step 7：报告输出 + 飞书推送

```
信息大图生成完成
- 风格：[风格名]（[风格ID]）
- 路径：[绝对路径]（3:4）
```

**自动推送飞书**（生成完成后自动执行）：
```bash
lark-cli im +messages-send --chat-id oc_77e72a8fa29356bf8942ac739bbd6dd5 --as bot --file [图片路径]
```

## 四、注意事项

1. **信息密度是核心** — 信息大图的价值在于数据密度，不是装饰
2. **Prompt 描述要具体** — 把模块要点转化为视觉描述融入 prompt
3. **默认 2K 分辨率** — 信息量大时可用 4K（设置 VF_IMAGE_SIZE=4K）
4. **比例固定 3:4** — 信息大图统一竖版，适配手机阅读
5. **一次生成为主** — 不满意时调整 prompt 重新生成
